## Why scaffold and not full coverage

`web-components-ui.md` §9.2 enumerates the full surface every Lit element MUST cover:

> Initial render for every meaningful property state (loading, empty, populated, error). Property reactivity. Event emission via real DOM events. Config-layer merge (data-* + updateConfig + /config mock). `::part` and slot contracts. Accessibility smoke (axe). `@lit-labs/observers` lifecycle without leaks.

Full coverage = ~30 tests per element × 2 elements = 60 component tests. Scaffold-first lets us verify the harness wires up cleanly before investing in coverage volume.
